# PassGuard
비밀번호 보안 진단 및 안전한 비밀번호 생성 iOS 어플리케이션

## 1. 프로젝트 수행 목적

### 1.1 프로젝트 정의
사용자의 비밀번호 보안 수준을 진단하고, 안전한 비밀번호 생성을 도와주는 iOS 어플리케이션

### 1.2 프로젝트 배경
최근 개인정보 유출 사고가 빈번하게 발생하고 있으며, 그 원인의 상당수가 취약한 비밀번호에서 비롯된다. 그러나 대부분의 사용자는 자신의 비밀번호가 얼마나 위험한지 인지하지 못하고, "password123"이나 "qwerty" 같은 예측 가능한 비밀번호를 여러 서비스에서 반복 사용하고 있다.

이러한 문제를 해결하기 위해, 비밀번호의 강도를 객관적으로 측정하고 구체적인 개선 방안을 제시하며, 패턴이 없는 안전한 비밀번호를 자동으로 생성해주는 도구를 제공하려고 한다.

### 1.3 프로젝트 목표
- **보안 진단**: 비밀번호를 강도, 패턴, 재사용, 변경 주기 4가지 기준으로 분석하여 0~100점 점수화
- **패턴 탐지**: 사전 단어, Leet Speak 치환, 키보드 패턴, 순차 문자 등 8가지 취약 패턴 감지
- **안전한 생성**: 감지된 패턴이 없는 비밀번호를 자동으로 생성
- **보안 저장**: SHA-256 해시 + iOS Keychain으로 비밀번호 안전 보관

## 2. 프로젝트 개요

### 2.1 프로젝트 설명
PassGuard는 사용자가 등록한 각 계정의 비밀번호를 종합적으로 분석하여 보안 점수를 산출한다.

**보안 점수 산정 기준:**

| 항목 | 점수 범위 | 설명 |
|------|-----------|------|
| 강도 점수 | 0 ~ 60점 | 엔트로피(log₂(문자종류) × 길이) 기반 계산 |
| 변경 주기 | 0 ~ 15점 | 90일 이내 +15 / 180일 이내 +8 / 초과 +0 |
| 패턴 감점 | 최대 -20점 | 사전단어, Leet치환, 키보드패턴, 순차/반복문자 등 |
| 재사용 감점 | -15점 | 다른 계정과 동일 비밀번호 사용 시 |

**패턴 감지 항목:**
- 사전 단어 탐지 (100개 이상 단어 DB)
- Leet Speak 치환 감지 (P@ssw0rd → password)
- 키보드 패턴 감지 (qwerty, asdf 등)
- 순차/역순 문자 감지 (abc, 321)
- 반복 문자 감지 (aaa)
- 날짜 형식 감지 (2024, 0101)
- 문자 분포 편향 감지 (한 종류 70% 이상)
- 단어+숫자 조합 감지 (hello123)

홈 대시보드에서 전체 보안 현황을 한눈에 파악할 수 있으며, 계정별 상세 진단 화면에서는 구체적인 경고와 보완 제안을 제공한다. 비밀번호 생성기는 최대 50회 재시도를 통해 패턴이 없는 안전한 비밀번호만을 생성한다.

### 2.2 프로젝트 구조

```
FinalPJ/
├── FinalPJApp.swift                  # 앱 진입점
├── ContentView.swift                 # 인증 분기 + TabView (4탭)
├── Item.swift                        # Account 데이터 모델 (SwiftData)
├── KeychainManager.swift             # Keychain CRUD + SHA-256 해싱
├── PasswordAnalyzer.swift            # 비밀번호 분석 엔진 (8종 패턴 탐지)
├── AuthViewModel.swift               # 로그인 / 회원가입 / 탈퇴
├── AccountViewModel.swift            # 계정 CRUD + 재사용 감지
├── PasswordGeneratorViewModel.swift  # 비밀번호 생성기 로직
├── SettingsViewModel.swift           # 설정 + JSON 내보내기/가져오기
├── LoginView.swift                   # 로그인 / 회원가입 화면
├── HomeDashboardView.swift           # 홈 대시보드 (원형 게이지 + 요약 카드)
├── AccountListView.swift             # 계정 목록 / 상세 진단 / 추가
├── PasswordGeneratorView.swift       # 비밀번호 생성기 화면
└── SettingsView.swift                # 설정 화면
```

### 2.4 기대효과
- 사용자가 자신의 비밀번호 보안 수준을 객관적인 점수로 인식하여 보안 의식이 향상된다.
- 취약한 패턴과 구체적인 개선 방안을 제시하여 실질적인 비밀번호 강화를 유도한다.
- 안전한 비밀번호 자동 생성으로 복잡한 비밀번호를 직접 고민할 필요가 없어진다.
- 계정별 재사용 감지를 통해 하나의 비밀번호가 유출되어도 다른 계정이 연쇄 피해를 받는 것을 예방한다.

### 2.5 관련 기술

| 구분 | 설명 |
|------|------|
| SHA-256 | 미국 국가안보국(NSA)이 설계한 암호화 해시 함수로, 임의의 길이 데이터를 256비트 고정 길이 해시값으로 변환한다. 단방향 함수로 해시값에서 원본 데이터를 복원할 수 없어 비밀번호 저장에 널리 사용된다. |
| iOS Keychain | Apple이 제공하는 보안 저장소로, 하드웨어 수준의 암호화를 통해 비밀번호, 토큰 등 민감한 데이터를 안전하게 저장한다. 앱이 삭제되어도 데이터가 유지되며, 기기 잠금 상태에서는 접근이 차단된다. |
| 엔트로피 (Entropy) | 정보 이론에서 정보의 불확실성을 측정하는 단위로, 비밀번호 보안에서는 log₂(문자종류) × 길이로 계산한다. 엔트로피가 높을수록 무차별 대입 공격에 대한 저항력이 강하다. |
| SwiftData | Apple이 2023년 WWDC에서 발표한 데이터 영속성 프레임워크로, @Model 매크로를 통해 Swift 클래스를 자동으로 데이터베이스 모델로 변환한다. Core Data를 대체하는 차세대 기술이다. |
| MVVM | Model-View-ViewModel 아키텍처 패턴으로, UI(View)와 비즈니스 로직(ViewModel)을 분리하여 코드의 유지보수성과 테스트 용이성을 높이는 설계 방식이다. |
